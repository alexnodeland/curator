# Personal search engines — a field survey

Every few years the same idea resurfaces: index everything I read,
wrote, and saved, then search it like the web. The graveyard is large
because most attempts couple three things that want to stay separate —
capture, storage, and retrieval.

The designs that age well share a shape: plain files as the canonical
store (they outlive every app), a derived index nobody is afraid to
delete, and retrieval as a thin tool over both. Agents slot into that
shape naturally as one more retrieval client, provided writes go
through a review gate instead of straight into the store.
